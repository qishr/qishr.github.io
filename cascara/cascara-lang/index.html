    Cascara Language Processors API

    Modules io.github.qishr.cascara.lang.*

    The Cascara Ecosystem provides a language-agnostic lexing, parsing, emitting, serializing, and schema APIs.

    Tokenizers, parsers, emitters, and serializers:

    • io.github.qishr.cascara.lang.css

    • io.github.qishr.cascara.lang.markdown
    • io.github.qishr.cascara.lang.xml

    Additionally, and the JSONPath module provides language-agnostic querying:

    • io.github.qishr.cascara.lang.jsonpath

    Reporting

    Package io.github.qishr.cascara.common.diagnostic

    The Reporter provides a standardized way for Tokenizers, Parsers, and Schema Validators to communicate issues without throwing hard exceptions, allowing for multi-error reporting.

    void errorAt(int line, int column, URI uri, Object... m);

    Reporter setLogger(ReportWriter logger);

    Emitter API

    Package io.github.qishr.cascara.common.lang.processor

    Emitters provide a low-level, write-only stream for producing structured text. They are responsible for:

    Indentation Management: Tracking depth and applying the indentSize from options.

    Separation Logic: Handling the specific characters between keys, values, and list items.

    Escaping: Ensuring scalars are safely represented (quotes, escaping, etc.).

    public interface Emitter extends Processor {
    void emitScalar(String value);
    void emitMapStart();
    void emitMapEnd();
    void emitSequenceStart();
    void emitSequenceEnd();
    void emitPropertySeparator();
    void emitItemSeparator();
    void emitNewLine();
    void indent();
    void dedent();
    String getOutput();
}
